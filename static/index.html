📊 Ray Data vs Spark SQL 对比

5 个核心区别点

🟢 Ray Data

  • GPU 原生支持 - 原生支持 CPU + GPU 异构计算
  • 非结构化数据 - 擅长处理视频、图像、文本等多模态数据
  • AI 工作负载优先 - 面向 LLM、深度学习、批量推理优化
  • 细粒度并行 - Actor 模型，支持任意 Python 程序扩展
  • 流式 GPU 预取 - 最大化 GPU 利用率

🟠 Spark SQL

  • CPU 为主 - 主要面向 CPU 环境，GPU 支持有限
  • 结构化数据 - SQL 和 DataFrame 优化用于结构化数据
  • 传统 BI 分析 - 面向 ETL、报表、经典机器学习
  • 数据并行模型 - 适合数据工作负载，灵活性有限
  • 批处理优化 - Catalyst 优化器针对 SQL 查询优化

📋 详细对比表

对比维度 Ray Data Spark SQL

import ray.data
import pandas as pd

# 创建数据集
df = pd.DataFrame({"value": range(10000)})
ds = ray.data.from_pandas(df)

# Map 操作
ds = ds.map(lambda x: {"value": x["value"] * 2 + 1})

# Filter 操作
ds = ds.filter(lambda x: x["value"] > 1000)

# FlatMap 操作
ds = ds.flat_map(lambda x: [{"value": x["value"]}, {"value": x["value"] * 2}, {"value": x["value"] * 3}])

# 获取结果
result = ds.count()

import ray.data

# 创建大规模数据集
ds = ray.data.range(50000)

# 流式处理
def process_batch(batch):
    return pd.DataFrame({"value": [x*2 for x in batch]})

# 配置预取
ds = ds.map_batches(
    process_batch,
    batch_size=100,
    prefetch_blocks=1  # GPU 预取
)

# 聚合统计
result = ds.sum(on="value")

✨ 细粒度任务并行 ✨ 有状态 Actor ✨ 流式 DAG 调度
动画自动循环 · 点击切换场景
🟢 Ray Data：任务粒度细到单个 Python 函数

同时跑成百上千个轻量 task，调度器按需分发，慢 worker 不会拖累全局；任意 Python 代码 + Actor 都能并行（不只是 SQL/DataFrame）。

🟠 Spark：以 partition 为单位整块迁移

partition 颗粒大且 Stage 间有 barrier；想跑非 SQL 代码必须套 UDF，跨 stage 必须等所有 partition 完成才能进入下一阶段。

🟢 Actor 一次初始化，复用资源

大模型权重、GPU 缓存、数据库连接池、向量索引……全部只在首次调用时加载，后续调用毫秒级返回。AI 推理服务最爱的模式。

🟠 函数式无状态 = 每次冷启动

Spark 的 UDF / Map 函数无法持有状态，模型加载、GPU 上下文、连接建立每次都要重来一遍，难以做高性能 LLM 推理。

🟢 流式 DAG · 块级流水线并行

数据按 block 切分，每个 block 完成上游算子立刻进入下游；下游 GPU 等待时上游已经预取好下一批，多阶段同时工作 → GPU 利用率最大化。

🟠 Stage Barrier · 全员对齐

Catalyst 把执行图切成 Stage，每个 Stage 必须全部完成（含 shuffle）才能进入下一个；GPU 算子常被慢的 IO/CPU 阶段拖到空转。

核心 API（对应上图）

import ray

# ① 细粒度任务：任意 Python 函数 → 远程任务
@ray.remote
def score(x): return model_predict(x)

futures = [score.remote(x) for x in dataset]   # 上千个并行任务
results = ray.get(futures)

# ② 有状态 Actor：模型 / 连接 / 缓存只加载一次
@ray.remote(num_gpus=1)
class LLMWorker:
    def __init__(self):
        self.model = load_llm()      # 仅启动时加载

    def infer(self, batch):
        return self.model(batch)     # 后续调用复用 GPU 状态

actor = LLMWorker.remote()
[actor.infer.remote(b) for b in batches]

# ③ 流式 DAG：block 级流水线，下游 GPU 永不空转
ray.data.read_parquet("s3://...") \
    .map(decode) \
    .filter(quality_ok) \
    .map_batches(LLMWorker, num_gpus=1, concurrency=4) \
    .write_parquet("s3://...")
